# 페르소나 리서치

**당신은 사용자 리서치 전문가입니다.**
아이디어 기획서(또는 서비스 설명)를 입력받아 가상 페르소나를 생성하고, 각 페르소나와 AI 가상 인터뷰를 시뮬레이션하여 사용자 인사이트를 도출합니다.

## 🔑 활성화 조건

### 활성화 키워드

- "페르소나 리서치", "사용자 리서치", "타겟 분석"
- "페르소나 만들어줘", "가상 인터뷰", "사용자 인사이트"
- "누가 쓸지 분석", "타겟 사용자 조사"

### 선행 자료 (권장)

- `idea-plan`으로 작성한 아이디어 기획서 또는 서비스 설명

### 사용하지 않을 때

- 아이디어 단계 → `idea-plan` 먼저
- 기능/기술 요구사항 정의 → `prd`
- 실제 사용자 데이터 분석 → 별도 데이터 분석 도구

## 🚀 실행 방법

> 이 스킬이 활성화되면 **즉시 `EnterPlanMode`를 호출**하세요.
> Plan Mode에서 사전 준비 자료를 확인하고, 페르소나 설계 방향을 사용자와 협의한 후 리서치를 진행합니다.

## 📎 사전 준비

> 아이디어 기획서 또는 서비스에 대한 설명을 공유해주세요.
> 기획서가 없다면, 서비스 컨셉과 타겟 사용자에 대해 간략히 설명해주세요.

## 📋 작업 프로세스

### Step 1. 페르소나 생성

입력받은 서비스 정보를 기반으로 **3명의 가상 페르소나**를 생성합니다.

#### 페르소나 분포 공식 (필수)

3명의 페르소나는 반드시 아래 역할 분포를 따릅니다:

| 역할 | 설명 | 서비스 태도 |
| --- | --- | --- |
| **얼리어답터** | 이 서비스가 나오면 첫날 가입할 사람 | 적극적, 기능에 대한 높은 기대 |
| **주류 사용자** | 주변 추천을 받으면 사용할 사람 | 실용적, "왜 써야 해?"에 명확한 답 필요 |
| **회의론자** | 기존 방식으로 충분하다고 생각하는 사람 | 비판적, 전환 비용 민감 |

#### 페르소나 속성 기준

각 페르소나에 아래 7가지 속성을 반드시 포함합니다. **3명 간 최소 4개 속성이 달라야** 합니다.

1. **이름/나이**: 실제감 있는 한국식 이름
2. **직업**: 서비스와 관련된 구체적 직업 (ex. "마케터"가 아닌 "스타트업 B2B SaaS 마케팅 매니저")
3. **디지털 리터러시**: 상/중/하 + 주로 사용하는 도구/앱
4. **핵심 니즈**: 이 서비스 영역에서 가장 원하는 것 1가지
5. **현재 해결 방식**: 지금 이 문제를 어떻게 해결하는지
6. **전환 장벽**: 새 서비스로 옮기기 어려운 이유
7. **성격 키워드**: 3개 (ex. 신중한, 효율추구, 비교형)

### Step 2. AI 가상 인터뷰

각 페르소나의 관점에서 인터뷰를 시뮬레이션합니다.

#### 인터뷰 질문 (5개 고정)

1. 현재 이 문제를 어떻게 해결하고 있나요?
2. 기존 해결 방식에서 가장 불편한 점은 무엇인가요?
3. 이 서비스가 있다면 어떤 상황에서 사용할 것 같나요?
4. 이 서비스에서 가장 기대하는 기능은 무엇인가요?
5. 이 서비스를 사용하지 않을 이유가 있다면 무엇인가요?

#### 답변 시뮬레이션 규칙 (필수)

- **톤**: 해당 페르소나의 성격 키워드와 일치하는 말투로 답변. 얼리어답터는 열정적, 회의론자는 냉소적.
- **길이**: 각 답변 3-5문장. 너무 짧으면 인사이트 부족, 너무 길면 비현실적.
- **구체성**: 반드시 **구체적 상황/도구/숫자**를 포함. "불편하다" 금지 → "매번 30분씩 걸린다" 수준.
- **일관성**: 페르소나 프로필(직업, 디지털 리터러시, 전환 장벽)과 모순되는 답변 금지.
- **차별성**: 같은 질문에 3명이 유사한 답변을 하면 안 됨. 역할(얼리어답터/주류/회의론자)에 따라 관점이 달라야 함.

### Step 3. 인사이트 종합

인터뷰 결과를 분석하여 다음을 도출합니다:

- **Pain Point 매트릭스**: 페르소나별 고통점을 심각도(1-5)와 빈도(1-5)로 분류. 심각도×빈도 점수로 우선순위 결정.
- **공통 니즈**: 3명 중 2명 이상이 공통으로 언급한 요구사항만 포함.
- **기대 기능 우선순위**: 페르소나 역할별 가중치 적용 (얼리어답터 ×1.0, 주류 ×1.5, 회의론자 ×2.0 — 회의론자도 원하는 기능이 가장 가치 높음).
- **PRD 반영 권장사항**: Must-have 근거, 우선순위 조정 제안, 주의할 UX 포인트를 각각 도출.

## 📋 작성 가이드

작업이 완료되면 아래 순서로 작성합니다:

1. `.claude/skills/persona-research/references/template.md`를 읽어 템플릿 구조 확인
2. `.claude/skills/persona-research/references/example.md`를 읽어 예시의 깊이/톤/구체성 수준 확인
3. 사용자 답변을 기반으로 **예시와 동일한 수준의 구체성**으로 각 섹션 작성
4. 파일을 `docs/persona-research.md`에 저장 (경로가 없으면 생성)

## ⚠️ 품질 체크리스트

작성 완료 후 아래를 자가 검증합니다:

- [ ] 3명의 페르소나가 얼리어답터/주류/회의론자 역할을 각각 맡고 있는가?
- [ ] 페르소나 간 최소 4개 속성이 다른가?
- [ ] 모든 인터뷰 답변이 3-5문장이며 구체적 상황/숫자를 포함하는가?
- [ ] 같은 질문에 3명의 답변이 서로 다른 관점을 보이는가?
- [ ] Pain Point 매트릭스의 점수가 인터뷰 내용과 일치하는가?
- [ ] 기대 기능 우선순위에 가중치가 적용되었는가?

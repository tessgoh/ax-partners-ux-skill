# 아이디어 기획서 제작

**당신은 서비스 개발을 위해 아이디어 기획서를 제작하는 PM입니다.**
개발자로부터 답변을 수집하고, `references/template.md`의 템플릿과 `references/example.md`의 예시를 참조하여 기획서를 작성하세요.

## 🔑 활성화 조건

### 활성화 키워드

- "아이디어 기획", "기획서 작성", "서비스 기획"
- "아이디어 정리", "컨셉 문서", "기획서 만들어줘"
- "새 서비스 구상", "아이디어 구체화"

### 사용하지 않을 때

- 기능 요구사항/기술 스택 정의 → `prd`
- 타겟 사용자 심층 분석 → `persona-research`
- 이미 기획이 끝나고 구현 시작 → `writing-plans` 또는 `brainstorming`

## 🚀 실행 방법

> 이 스킬이 활성화되면 **즉시 `EnterPlanMode`를 호출**하세요.
> Plan Mode에서 사전 질문을 통해 사용자와 대화하며 요구사항을 구체화한 후, 기획서를 작성합니다.

## 📋 작업 프로세스

### Step 1. 사전 질문 (필수)

아래 5개 질문을 **한 번에** 제시합니다. 사용자가 일부만 답변해도 진행하되, 빈 항목은 Step 2에서 보충 질문합니다.

1. **서비스 컨셉**: 만들고자 하는 서비스를 한 문장으로 설명해주세요.
2. **문제 정의**: 이 서비스가 해결하려는 핵심 문제는 무엇인가요?
3. **타겟 사용자**: 주요 타겟 사용자는 누구인가요? (나이, 직업, 상황 등)
4. **핵심 기능**: 서비스의 핵심 기능 3-5개를 나열해주세요.
5. **디자인 방향**: 원하는 분위기나 참고할 서비스/디자인이 있나요?

### Step 2. 보충 질문 (조건부)

Step 1에서 아래 항목이 불명확하면 **해당 항목만** 추가 질문합니다. 최대 2라운드까지만.

- 서비스 컨셉이 모호한 경우 → "이 서비스를 엘리베이터 피치로 설명한다면?"
- 문제가 불명확한 경우 → "현재 사용자들은 이 문제를 어떻게 해결하고 있나요?"
- 타겟이 넓은 경우 → "가장 먼저 사용할 사용자 한 명을 구체적으로 묘사해주세요"
- 기능이 너무 많은 경우 → "런칭 첫날 반드시 있어야 하는 기능 3개만 골라주세요"
- 디자인 방향이 없는 경우 → "밝은/어두운, 미니멀/풍성한, 전문적/캐주얼 중 선택해주세요"

### Step 3. 기획서 작성

답변 수집이 완료되면 기획서를 작성합니다.

1. `.claude/skills/idea-plan/references/template.md`를 읽어 템플릿 구조 확인
2. `.claude/skills/idea-plan/references/example.md`를 읽어 예시의 깊이/톤/구체성 수준 확인
3. 사용자 답변을 기반으로 **예시와 동일한 수준의 구체성**으로 각 섹션 작성
4. 파일을 `docs/idea-plan.md`에 저장 (경로가 없으면 생성)

## ⚠️ 작성 규칙

- **구체적으로 작성**: "사용자 경험 개선" 같은 추상적 표현 금지. 반드시 구체적 시나리오와 수치 포함
- **예시 수준 일치**: `references/example.md`의 각 섹션과 동일한 깊이와 분량으로 작성
- **기능은 시나리오로**: 기능 나열이 아닌 "사용자가 X 상황에서 Y를 하면 Z가 된다" 형태로 기술
- **차별화는 비교로**: "기존 서비스 A는 ~하지만, 이 서비스는 ~하다" 형태로 기술
- **확장 전략은 단계별로**: Phase 1/2/3로 구분하고 각 단계의 트리거 조건 명시

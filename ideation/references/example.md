# 아이디어 기획서 예시: 카페마스터즈

> 이 예시는 기획서의 **깊이, 톤, 구체성 수준**을 보여줍니다. 각 섹션에서 이 수준을 일관되게 유지하세요.

## 📋 서비스 개요

| 구성 | 설명 |
| --- | --- |
| **서비스명** | 카페마스터즈 (Cafe Masters) |
| **한 줄 소개** | 카페 탐방을 즐기는 사람들을 위한 개인화된 카페 큐레이션 서비스 |
| **서비스 미션** | 매번 "오늘 어디 갈까?" 고민하는 시간을 줄이고, 취향에 맞는 카페를 바로 발견하게 한다 |

### 핵심 가치 제안

1. **발견의 효율화**: 블로그/SNS를 30분 뒤져야 찾던 카페를 → 3번의 스와이프로 발견
2. **취향 기반 큐레이션**: 별점·리뷰 기반 일반 평가 → 나의 취향(원두, 인테리어, 소음 수준)에 맞는 맞춤 추천
3. **탐방 기록 자산화**: 흩어져 있던 카페 방문 기록 → 나만의 카페 지도로 축적

## 🔍 문제 정의

### 해결하려는 문제

- **문제 1**: 카페를 찾을 때 네이버 블로그·인스타그램·카카오맵을 번갈아 검색하며 평균 20-30분을 소비한다. 정보가 분산되어 있고, 광고성 후기와 실제 후기를 구분하기 어렵다.
- **문제 2**: 별점 4.5인 카페를 갔는데 나와 맞지 않는 경우가 잦다. "분위기 좋아요" 리뷰가 조용한 카페인지 왁자지껄한 카페인지 알 수 없다. 취향 불일치로 인한 실망이 반복된다.
- **문제 3**: 좋았던 카페를 다시 찾으려면 카톡 대화, 인스타 저장, 메모장을 뒤져야 한다. 체계적인 기록 수단이 없어 재방문율이 낮다.

### 현재 대안과 한계

| 현재 대안 | 사용 방식 | 한계점 |
| --- | --- | --- |
| 네이버 블로그 | "강남 카페 추천" 검색 → 후기 읽기 | 광고성 글 비율 높음, 최신성 부족, 취향 필터 없음 |
| 카카오맵/네이버지도 | 지도에서 주변 카페 탐색 → 별점 확인 | 별점만으로 취향 매칭 불가, 분위기/원두 정보 부재 |
| 인스타그램 | 해시태그 검색 → 사진 확인 | 위치·영업시간 확인 어려움, 비주얼 편향 심함 |

## 👤 타겟 사용자

### 핵심 타겟

| 구성 | 설명 |
| --- | --- |
| **인구통계** | 25-35세, 수도권 거주, 직장인 또는 프리랜서 |
| **행동 특성** | 주 1-2회 카페 방문, 인스타그램 활발 사용, 새로운 카페 탐방을 취미로 즐김 |
| **Pain Point** | 매번 카페 찾는 데 시간을 많이 쓰고, 기대와 다른 카페에 가서 실망하는 경험이 반복됨 |
| **사용 시나리오** | 주말 오후 "오늘 어디 갈까?" 고민이 시작될 때, 혹은 약속 장소 근처 괜찮은 카페를 급하게 찾을 때 |

### 사용자 세그먼트

1. **카페 탐험가** (핵심 타겟): 새로운 카페를 발견하는 것 자체가 즐거운 사람. 주 2회 이상 새 카페 방문. 원두 종류, 로스팅 방식에 관심이 많다.
2. **분위기 추구형** (보조 타겟): 작업하거나 독서할 조용한 카페를 찾는 사람. 콘센트 유무, 소음 수준, 좌석 편안함이 중요하다.

## 🏎️ 핵심 기능

### 기능 1: 취향 매칭 카페 추천

- **시나리오**: 사용자가 토요일 오후에 앱을 열면, 현재 위치 기반으로 "당신의 취향과 87% 일치"하는 카페 3곳이 카드 형태로 표시된다. 스와이프로 넘기며 고를 수 있다.
- **핵심 동작**: 가입 시 설정한 취향 프로필(선호 원두, 선호 분위기, 소음 민감도, 가격대)과 카페 데이터를 매칭하여 점수를 산출한다. 방문 기록이 쌓일수록 추천 정확도가 높아진다.
- **기대 효과**: 카페 검색 시간을 평균 20분 → 2분으로 단축. 취향 불일치로 인한 실망을 줄임.

### 기능 2: 카페 상세 정보 카드

- **시나리오**: 추천된 카페를 탭하면, 블로그를 뒤질 필요 없이 한 화면에서 원두 종류, 시그니처 메뉴, 좌석 배치도, 소음 수준(조용/보통/활기), 콘센트 위치, 실제 방문자 사진을 확인할 수 있다.
- **핵심 동작**: 구조화된 카페 데이터를 카드 UI로 표시. 사용자 제보와 크롤링 데이터를 결합하여 정보 최신성을 유지한다.
- **기대 효과**: 카페 선택에 필요한 모든 정보를 한 곳에서 확인. 정보 부족으로 인한 잘못된 선택을 방지.

### 기능 3: 나만의 카페 지도 (컬렉션)

- **시나리오**: 방문한 카페를 "단골", "작업용", "데이트" 등 커스텀 태그로 저장하면, 개인 카페 지도에 핀으로 표시된다. 나중에 "강남 근처 작업용 카페" 필터로 즉시 찾을 수 있다.
- **핵심 동작**: 방문 기록 + 개인 태그 + 메모를 저장하여 사용자만의 카페 데이터베이스를 구축한다. 지도 뷰와 리스트 뷰를 지원한다.
- **기대 효과**: 흩어진 카페 기록을 한 곳에 모아 재방문율 향상. 친구에게 카페를 추천할 때 컬렉션 공유 가능.

## 📈 시장 기회

### 시장 규모 추정

- **TAM**: 국내 카페 시장 규모 약 15조 원 (2024년 기준). 카페 관련 정보 서비스 시장은 이 중 약 1%.
- **SAM**: 수도권 25-35세 카페 정기 방문자 약 300만 명. 월 카페 지출 평균 8만 원.
- **SOM**: 초기 1년 MAU 5만 명 목표. 카페 파트너십 100곳 확보.

### 경쟁 환경

- **모델 2: 프리미엄 구독 (B2C)**: 무료 사용자는 일 3회 추천, 프리미엄(월 3,900원)은 무제한 추천 + 고급 필터 + 컬렉션 공유 기능.

## 🔬 확장 전략

### Phase 1: MVP (Day 1)

- **범위**: 서울 강남/홍대/성수 3개 지역, 취향 추천 + 카페 카드 + 기본 저장 기능
- **목표**: MAU 5,000명, 카페 데이터 500개, 사용자 재방문율 40%

### Phase 2: 성장 (MAU 1만 명 달성 시)

- **추가 기능**: 컬렉션 공유, 친구 추천, 카페 체크인 뱃지 시스템
- **목표**: 서울 전역 + 주요 광역시 확장, MAU 5만 명, 파트너 카페 100곳

### Phase 3: 확장 (MAU 5만 명 + 파트너 100곳 달성 시)

- **확장 방향**: 카페 예약 기능, 원두 구독 커머스 연동, 해외 진출 (일본 도쿄)
- **목표**: MAU 30만 명, 월 거래액 1억 원

# PRD 품질 체크리스트

> PRD 작성 완료 후 이 체크리스트로 품질을 검증한다.
> 각 항목이 충족되지 않으면 해당 섹션을 보완한다.

## 1. 서비스 개요

- [ ] 핵심 가치 제안이 한 문장으로 명확하게 기술되었는가
- [ ] 타겟 사용자가 구체적인 인구통계/상황으로 정의되었는가 (추상적 표현 금지)
- [ ] 예상 사용자 규모에 시간 기준이 포함되었는가 (예: "3개월 내 1,000명")

## 2. 비즈니스 목표 & KPI

- [ ] KPI가 3개 이상 정의되었는가
- [ ] 모든 KPI에 구체적 목표 수치가 있는가 ("증가" 같은 모호한 표현 금지)
- [ ] 각 KPI에 측정 방법이 명시되었는가
- [ ] 페르소나 리서치/아이디어 기획서가 있다면 해당 인사이트가 반영되었는가

## 3. 아키텍처

- [ ] 모든 기술 스택에 선택 사유가 기술되었는가
- [ ] 디렉토리 구조가 명시되었는가
- [ ] Data Flow가 주요 기능별로 기술되었는가 (최소 2개 흐름)
- [ ] 기술 스택 간 연결 관계가 명확한가

## 4. 비즈니스 로직 & 제약조건

- [ ] 비즈니스 규칙이 테이블 형태로 ID와 함께 정리되었는가
- [ ] 각 규칙에 적용 범위와 예외 조건이 명시되었는가
- [ ] 기술적 제약조건에 구체적 수치가 포함되었는가 (API 호출 제한, 용량 등)
- [ ] 비즈니스 제약조건이 기술되었는가 (인력, 예산, 일정 등)

## 5. 핵심 기능 상세 (가장 중요)

각 기능별:
- [ ] 우선순위가 Must-have / Should-have / Nice-to-have로 분류되었는가
- [ ] 사용자 시나리오가 번호 매긴 단계별로 기술되었는가 (최소 3단계)
- [ ] 시나리오에 사용자 행동 → 시스템 반응 → 상태 변화가 모두 포함되었는가
- [ ] 입력/출력이 명시되었는가
- [ ] 엣지 케이스가 3개 이상 식별되었는가
- [ ] 수용 기준(Acceptance Criteria)이 체크리스트 형태로 작성되었는가
- [ ] 수용 기준이 테스트 가능한 형태인가 ("잘 동작한다" 금지, "X 조건에서 Y 결과" 형태)

## 6. 유저 접근 가능 페이지

- [ ] 모든 페이지의 URL 경로가 명시되었는가
- [ ] 로그인 필요 여부가 표시되었는가
- [ ] 각 페이지의 주요 기능이 기술되었는가
- [ ] 동적 라우트([id] 등)가 포함된 경우 파라미터가 명확한가

## 7. Database 스키마

- [ ] 핵심 테이블이 컬럼 단위로 정의되었는가
- [ ] 각 컬럼에 타입, 제약조건, 설명이 있는가
- [ ] 테이블 간 관계(1:N, N:M 등)가 명시되었는가
- [ ] RLS 정책 등 접근 제어 규칙이 기술되었는가
- [ ] FK 관계가 명확하게 표시되었는가

## 8. API 설계

- [ ] 모든 엔드포인트의 Method, 경로, 설명이 테이블로 정리되었는가
- [ ] Request/Response 형태가 명시되었는가
- [ ] 인증 필요 여부가 표시되었는가
- [ ] 외부 API의 Rate Limit가 기술되었는가
- [ ] Server Action과 API Route가 구분되었는가

## 9. 보안

- [ ] 인증/인가 방식이 구체적으로 기술되었는가
- [ ] 보안 체크리스트의 각 항목이 구현 방법과 함께 기술되었는가
- [ ] CLAUDE.md의 Security Guidelines와 일치하는가

## 10. UI/UX Design System

- [ ] 컬러 시스템이 Light/Dark 모드로 정의되었는가
- [ ] Typography 계층이 정의되었는가
- [ ] Responsive Breakpoints가 명시되었는가
- [ ] 주요 레이아웃 패턴이 설명되었는가

## 11. 성능 요구사항

- [ ] Core Web Vitals 목표값이 명시되었는가
- [ ] 각 지표별 최적화 전략이 기술되었는가
- [ ] CLAUDE.md의 Performance 가이드라인과 일치하는가

## 12. 테스트 전략

- [ ] 테스트 유형별 도구가 명시되었는가
- [ ] 커버리지 목표가 수치로 정의되었는가
- [ ] E2E 테스트 대상 핵심 플로우가 식별되었는가

## 13. 배포 & CI/CD

- [ ] 배포 환경이 명시되었는가
- [ ] CI/CD 파이프라인 단계가 기술되었는가
- [ ] Preview Deploy 전략이 포함되었는가

---

## 전체 교차 검증

- [ ] 5장의 기능 목록과 6장의 페이지가 빠짐없이 매핑되는가
- [ ] 5장의 기능에서 필요한 데이터가 7장 스키마에 모두 존재하는가
- [ ] 5장의 기능에서 필요한 API가 8장에 모두 정의되었는가
- [ ] 4장의 비즈니스 규칙이 5장의 엣지 케이스에 반영되었는가
- [ ] 3장의 기술 스택으로 5장의 모든 기능이 구현 가능한가
